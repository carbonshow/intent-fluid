# Test Suite: Iterative Intent Validation

This directory contains three distinct PRD (Product Requirement Document) samples designed to stress-test the **Intent-Fluid** skill. Each sample targets a specific dimension of the AI's ability to drive autonomous iteration and intent discovery.

---

## 📂 Test Case 01: Functional Logic & Edge Cases

**File:** `01_Subscription_Tracker.md`

### 🏗️ Scenario

A standard, utility-based consumer application. The requirements are clear at the surface level but lack technical depth and operational rigor.

### 🎯 Test Objectives

* **Logic Completeness:** Does the Skill identify the need for "trial-to-paid" transition logic?
* **Data Integrity:** Does it catch the complexity of multi-currency handling and exchange rate updates?
* **UX Refinement:** Does it suggest specific notification channels (Email/Push/Webhooks) rather than just "notifying the user"?
* **Success Metric:** The output should evolve from a simple list to a robust functional specification ready for backend architecture.

## 🧪 Test Case 02: Conceptual Research & Innovation

**File:** `02_Aura_Link.md`

### 🏗️ Scenario

A high-level, research-oriented game design project involving "Embodied Intelligence." The requirements are abstract, academic, and lack a concrete implementation path.

### 🎯 Test Objectives

* **Creative Problem Solving:** Can the Skill propose a mathematical framework (e.g., Vector math or Gaussian distributions) to solve an abstract problem?
* **Autonomous Exploration:** Does it identify emergent risks, such as "Sentiment Decay" or "Recursive Feedback Loops"?
* **Domain Expertise Simulation:** Does it act like a Game Producer/Researcher by suggesting specific behavioral outputs (animations, pathfinding) rather than just dialogue?
* **Success Metric:** The output should provide a structured "Technical Roadmap" that bridges the gap between a wild idea and a buildable prototype.

## 🌫️ Test Case 03: Ambiguity & Intent Discovery

**File:** `03_Nexus_Burst.md`

### 🏗️ Scenario

A corporate marketing request filled with buzzwords ("viral," "synergy," "Z-Generation") and "flexible" constraints. This is a "Requirement Swamp" where the true intent is hidden behind vague corporate language.

### 🎯 Test Objectives

* **Proactive Interrogation:** Does the Skill stop to ask clarifying questions before generating a solution? (e.g., "What is the specific budget cap?" or "Define the primary KPI.")
* **De-Noising:** Can it filter out marketing jargon to find the core business problem (Growth Plateau)?
* **Strategic Alignment:** Does it demand brand guidelines and technical constraints before proposing an "AI-powered mechanic"?
* **Success Metric:** A "Hard Fail" is if the Skill generates a generic marketing plan. A "Hard Win" is if the Skill provides a list of **Critical Clarification Questions** to the user.

## 🚀 Test Case 04: Creative Synthesis & World-Building

**File:** `nova-logic.md`

### 🏗️ Scenario

A high-concept creative writing task. The goal is to architect and generate a multi-chapter Science Fiction novel that balances commercial viability with high-concept innovation.

### 🎯 Test Objectives

* **Market Alignment & Autonomy:** Does the Skill autonomously research and select a "viral-ready" sub-genre?
* **Creative Innovation:** Can it propose an "Anti-Cliché" hook and a "First-Principle" innovation that hasn't been widely explored?
* **Structural Coherence:** Does it provide a logical chapter-based structure with clear narrative functions (Exposition, Rising Action, etc.)?
* **Success Metric:** The output should be a coherent "Master Concept" and "Chapter Outline" that feels both fresh and commercially viable in Chinese.

## 📈 Evaluation Framework

To evaluate the iteration quality of the **Intent-Fluid** skill, use the following rubric:

| Level | Performance Description |
| :--- | :--- |
| **Level 1: Passive** | Simply re-formats the input PRD into a better layout. |
| **Level 2: Reactive** | Fills in obvious gaps (e.g., adding a "Delete" button to a list). |
| **Level 3: Proactive** | Identifies logic flaws and proposes industry-standard solutions. |
| **Level 4: Expert** | Challenges the user's assumptions, defines mathematical models, and uncovers hidden business intents. |

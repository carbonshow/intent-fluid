┌─────────────────────────────────────────────────────────────────────────────┐
│ EXECUTION TRACE & OBSERVABILITY (surge example)                             │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  TRACE FILE: {task_dir}/trace.jsonl (JSONL format, append mode)           │
│                                                                              │
│  Event Schema:                                                              │
│  {                                                                          │
│    "id": "evt_001",                  ← Auto-generated, zero-padded         │
│    "ts": "2026-03-27T18:00:20Z",     ← ISO 8601, UTC                      │
│    "skill": "surge",                                                        │
│    "type": "step_start",             ← step_start|step_end|agent_*|error  │
│    "step": "analyze",                ← Must be in trace.steps from SKILL  │
│    "round": 1,                       ← Iteration number (1-indexed)        │
│    "agent": "director",              ← director|subagent:<role>           │
│    "status": "executing",            ← executing|completed|failed          │
│    "status_display": "⚡ ...",        ← User-readable with emoji          │
│    "detail": {...},                  ← Key-value metadata                 │
│    "parent_id": "evt_000",           ← Forms event tree                   │
│    "tags": ["key:value"]             ← Skill-specific metadata            │
│  }                                                                          │
│                                                                              │
│  TRACE TOOLS:                                                               │
│  └─ bash scripts/trace.sh             (Emit event, returns event ID)       │
│  └─ bash scripts/trace-export.sh      (Export to Mermaid/markdown/ASCII)  │
│  └─ bash scripts/dashboard.sh         (Start/stop real-time viz)          │
│                                                                              │
│  OUTPUTS:                                                                   │
│  ├─ execution_dag.mmd                (Mermaid DAG visualization)           │
│  ├─ execution_summary.md             (Markdown table)                      │
│  ├─ execution trace visualization    (Real-time web dashboard)            │
│  └─ JSONL passthrough                (Datadog, Grafana, jq ingestion)     │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
